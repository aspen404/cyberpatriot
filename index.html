Week 1
Aperture Science — Ubuntu
Two days of easy wins to shake off the rust, then the real diagnostic. Build confidence first, then find the gaps.
—

About This Resource

What is it?

A community-made CyberPatriot practice virtual machine, themed around the Portal video game. It contains 100 hidden security vulnerabilities worth 250 points. A built-in scoring engine (called aeacus) runs in the background and updates your score in real-time as you fix things — like a scoreboard that responds immediately.

How does it work?

You download the VM image (~30GB), open it in VMware Workstation, and start working on the Ubuntu Linux system inside. The scoring engine is already installed. Every time you fix a real vulnerability — remove a bad user account, enable the firewall, patch a misconfigured service — your score ticks up. You can see your score live in a small window or browser tab.

Why this week?

You've competed before. You don't need a lecture — you need to know exactly where your gaps are right now, at this moment in summer 2026. This image is rated Round 2–3 difficulty (harder than Round 1), so it will find holes in your knowledge that Round 1 wouldn't. That's the point. High score isn't the goal. Information is.

What is "hardening"?

Hardening means making a system harder for an attacker to compromise — fixing weak passwords, disabling unused services, turning on the firewall, removing unauthorized users. That's what CyberPatriot's scoring engine checks. Every fix earns points. Breaking something required (a service the scenario says to keep running) loses points. There are also short text-based "forensics questions" worth bonus points — more on those below.

Why warm up first?

Aperture Science is Round 2–3 difficulty — it will find gaps even for experienced competitors. Starting cold on a hard image after months off is a fast path to a demoralizing Day 1. So: Days 1–2 use an Easy-rated image from Titan Turtles to get some wins, remember the workflow, and build buy-in. Then Aperture Science on Day 3 as the real diagnostic. Same tool, different weight class.

On Claude Code and competition day: This plan uses Claude throughout practice — that's intentional and great for learning. But CyberPatriot rules change every season. Preliminary and State rounds typically allow internet; National Finals do NOT. Some schools also restrict AI tools regardless of national rules. Build your skills assuming Claude won't be there on competition day. Every checklist item should have the actual command written out — not "ask Claude." Your Claude prompt file is for the edge cases, not the backbone.

This is the warmup image. Easy difficulty, Ubuntu 24, from Titan Turtles. The goal isn't to learn anything new — it's to shake off the months-off rust and remember what it feels like to score points. Hit the obvious stuff: bad users, firewall off, weak passwords, services that shouldn't be running. Let the score climb. Build the muscle memory back.

Day 5 — Debrief + Forensics

Forensics bonus questions — what they are: Competition images include text-based questions in the scenario file worth bonus points. They ask things like "What user last logged in?" or "What process is listening on port 4444?" or "Find the file modified in /tmp in the last hour." Key commands: last / lastlog (login history), ss -tulnp (what's listening), find /tmp -mtime -1 (recently modified files), cat ~/.bash_history (command history). Look for forensics questions in your scenario file — they're free points most teams miss.

Claude debrief exercise: Paste your gaps list and ask: "I'm prepping for CyberPatriot on an Ubuntu image. Here are the specific things I couldn't figure out: [list]. Walk me through each one." Take notes. Add the answers to your checklist.

OverTheWire: Bandit

About This Resource

What is it?

OverTheWire is a series of "wargames" — hacking challenges designed to teach specific skills. Bandit is the beginner one, and it's specifically about Linux. There are 34 levels. Each level gives you a clue, and you SSH into their server, use Linux commands to find a password, then use that password to unlock the next level. It's a game you play in your terminal.

How does it work?

You open a terminal and SSH into bandit.labs.overthewire.org. The first level is almost trivially easy — just reading a file. By level 10–15 you're combining commands with pipes, finding files by ownership or permission, decoding hidden data. No installation needed. Works on any laptop from anywhere — perfect for travel too.

Why this week?

Bandit teaches exactly what CyberPatriot needs: navigating the filesystem, finding files by permission or size or owner, understanding users and groups, reading protected files, piping commands together. It's disguised as a puzzle game, but every level is a real Linux skill. After 20 levels, you'll find things on the Aperture Science image you couldn't see before.

What to expect

Levels 0–8 will feel easy if you use Linux regularly. Levels 9–15 will require looking up a command or two. Levels 16–25 get into SSL, nmap, and other networking tools — that's stretch territory; don't get stuck there. The goal is to get to level 20 with solid understanding, not to blitz through all 34.

Day 5 — Map Back to CyberPatriot + SSH Hardening

SSH hardening — one of the most consistently scored Linux categories: Open /etc/ssh/sshd_config on your Ubuntu VM and add/change these settings:
PermitRootLogin no — prevents root login via SSH
Protocol 2 — disables the older, insecure SSH protocol
X11Forwarding no — blocks graphical app forwarding
MaxAuthTries 3 — limits login attempts
After editing: sudo systemctl restart ssh to apply. Add this as its own "SSH Config" section in your Linux checklist.

Claude exercise: Share your updated Linux checklist and ask: "I'm prepping for CyberPatriot on Ubuntu. Here's my current hardening checklist organized by category. What scoring categories am I missing or underdeveloped?" Merge the useful additions in.

What is it?

The Windows image from the same Aperture Science bundle — "Aperture Science Moon Base." Same concept: 100 vulnerabilities, 250 points, scored in real-time by aeacus. Windows 10, competition-style scenario. The auto-login is intentionally disabled because that's itself a vulnerability to find and fix.

Check the GitHub repo README for the default login credentials — they're listed there. Launch GUI tools by pressing Win+R and typing the command name (secpol.msc, gpedit.msc, etc.). Why this week?

Competition is split roughly evenly between Linux and Windows. You just spent two weeks leveling up your Linux game. Now apply the same systematic loop — diagnostic pass, identify gaps, build checklist, return run — to Windows. The logic is identical; the tools are different.

Companion: cypat.guide

cypat.guide is the best community-written CyberPatriot reference. Use its Windows section as a lookup while you work — not as a reading assignment. When you hit a wall on the image, open cypat.guide to see if the fix is documented there. Then add what you learn to your personal checklist in your own words.

About This Resource

What is it?

Under The Wire is the Windows equivalent of OverTheWire Bandit — a wargame where you SSH into a server and use PowerShell commands to solve challenges and unlock the next level. The "Century" series is the beginner track. It teaches the Windows command-line and PowerShell in the same gamified, level-by-level format that made Bandit useful for Linux.

How does it work?

SSH into their server (or use PSRemoting) to reach each level's environment. Each level gives a clue; you use PowerShell to find the password for the next level. No download needed — works from any terminal that can SSH. The challenges teach Get-ChildItem, Get-Content, Get-LocalUser, Get-Service, Get-Process, event log queries, and registry navigation — all directly relevant to Windows hardening.

Why this week?

You just did Week 3's Windows image with mostly GUI tools. Under The Wire builds the PowerShell fluency to check things that the GUI doesn't show cleanly: who's in what group, what services are running, what's in the registry, what processes are active. After this week, your Windows skills have the same depth your Linux skills got from Bandit. Then you return-run the Windows image to prove it.

What to expect

If you've never used PowerShell seriously, the syntax will feel odd at first — it's more verbose than bash. The key idea: PowerShell commands are Verb-Noun (Get-Service, Set-LocalUser, Remove-LocalGroupMember). Levels 1–8 of Century are approachable. Levels 9–15 are harder. Don't aim to finish Century entirely — the goal is the PowerShell vocabulary that translates directly to competition.

Week 5
Cisco Packet Tracer
The third leg of the competition. Learn to read and fix network configurations — routers, switches, VLANs, and basic security settings.
—

About This Resource

What is it?

Cisco Packet Tracer is a free network simulator made by Cisco. You drag routers, switches, and computers onto a canvas, connect them with virtual cables, and configure them using Cisco's IOS command-line interface (or a GUI). It simulates real Cisco hardware behavior without needing physical equipment. Free with a Cisco NetAcad account.

How does it work?

Download Packet Tracer, create a free NetAcad account to activate it. Open a blank canvas, add devices from the sidebar, connect them with the right cable types (straight-through vs. crossover — Packet Tracer handles this mostly automatically). Click a device to open its CLI, then type IOS commands to configure it. The simulation shows whether packets are flowing correctly.

Why this week?

CyberPatriot's networking challenge gives you a Packet Tracer file with a broken or insecure configuration. You read the existing config, spot what's wrong, and fix it. The skill isn't building networks from scratch — it's reading what's there and knowing what a secure configuration should look like. This week teaches you the vocabulary to do that.

What to expect

IOS has a mode hierarchy: User EXEC (>) → Privileged EXEC (#, reached with enable) → Global Config ((config)#, reached with configure terminal) → Interface Config ((config-if)#). Commands only work in the right mode — if you get "invalid command," you're probably in the wrong mode. Use ? after any partial command to see what's valid there. Use Claude for syntax this week — memorizing it isn't the goal, but show running-config is something you should be able to read fluently.

About This Resource

What is it?

Syned is a community site that hosts CyberPatriot-style practice VM images made by other competitors and coaches. Multiple Ubuntu and Windows images available, varying in difficulty and focus area. Different from Aperture Science: no themed scenario, no guaranteed answer key, more variety. Each image is 3–12GB.

How does it work?

Download an image from the site, import into VMware, boot it up. The scoring engine should be running. Work through it the same way you've been working through Aperture Science — read the scenario first, run your checklist, chase edge cases. The main difference: you haven't practiced on these images before. Unknown territory.

Why this week?

Aperture Science images are excellent, but by Week 6 you've run them multiple times. You're recognizing the specific vulnerabilities, not your own knowledge. Syned images reset that — you don't know what's in them. That unknown is what competition day feels like. The goal of Week 6 is to test your actual system against something you've never seen, then simulate a full 6-hour round.

What to expect + hardware note

Your checklist should handle 60–70% of points systematically. The remaining 30–40% is scenario-specific or harder vulnerabilities. That gap is normal. RAM note for the simulation (Day 4–5): running two VMs simultaneously needs at least 12–16GB RAM. If your machine has 8GB, run Ubuntu and Windows in alternating shifts instead of simultaneously — set 3 hours each rather than 2.5+2.5 at the same time.
